# 英语真题词频统计 WebApp — 实施计划（修订版）

## 一、项目概述

将 2000-2026 年考研英语真题 PDF 进行文本提取、题型解析、英文单词分词与词频统计，构建一个本地 Web 应用，支持：
- 词频统计结果展示与多维可组合筛选
- 单词在真题中的定位与上下文卡片展示
- 统计结果导出为 Excel（含筛选器）

---

## 二、技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| **PDF 文本提取** | Python + pdfplumber | 中文 PDF 兼容性好，能提取文本和位置 |
| **英文单词处理** | Python + NLTK（WordNet lemmatizer + POS tagger） | 词形还原、词性标注、不规则形式识别 |
| **命名实体识别** | NLTK + 自建专有名词词典 | 过滤人名/地名/机构名 |
| **数据处理** | Python + pandas | 强大的数据处理和统计分析 |
| **数据存储** | SQLite | 轻量、免安装，适合本地单机应用 |
| **后端 API** | Python + Flask | 轻量，适合本地 Web 应用 |
| **前端** | HTML + CSS + Vanilla JS | 无框架依赖，本地直接运行 |
| **Excel 导出** | openpyxl | 支持 AutoFilter 的 Excel 文件生成 |

> **注意：不需要 jieba（结巴分词）**——本项目只统计英文单词，不涉及中文分词。

## 三、PDF 内容提取规则（核心）

### 3.1 全局忽略项

以下内容在提取时**直接丢弃**，不进入任何统计：

| 忽略项 | 说明 |
|--------|------|
| 考试提示信息 | 各题型的大标题、Section 标题、Part 标题、Directions 行 |
| 水印 | `https://zhenti.burningvocabulary.cn` |
| 页码 | 页眉/页脚的页码数字 |
| 数字/题号 | 文中的数字编号、题号（如 `41.`、`[A]` 中的纯标记部分） |
| `A. B. C. D.` 选项标记 | 选项前的字母标记本身不计入 |

### 3.2 按年份区分的试卷结构

真题试卷结构随年份变化，需分别处理：

#### 结构 A：2000-2001 年（阅读 5 篇，翻译在 Part B）

```
Section I  完形填空
    ├── [忽略] 考试提示
    ├── ✅ 完形填空正文（统计，去掉挖空数字标记）
    └── ✅ 完形填空选项（统计，去掉题号及 A/B/C/D 标记）

Section II 阅读理解
    Part A
    ├── [忽略] 考试提示
    ├── Text 1~5
    │   ├── ✅ 阅读正文（统计）
    │   └── ✅ 阅读题目及选项（统计，去掉题号及 A/B/C/D）
    Part B（翻译）
    ├── [忽略] 考试提示
    └── ✅ 翻译正文（统计，去掉数字标记）

Section III 写作
    ├── [忽略] 考试提示
    └── ✅ 写作题目描述正文（统计）
```

#### 结构 B：2002-2004 年（阅读 4 篇，其余同结构 A）

与 2000-2001 基本相同，唯一区别：**阅读只有 4 篇**（Text 1~4）。

#### 结构 C：2005-2026 年（阅读 4 篇，Part B 变为新题型）

```
Section I  完形填空（同结构 A）

Section II 阅读理解
    Part A（阅读 4 篇，同结构 B）
    Part B（7选5 / 排序 / 选标题）  ← 2005 年起新增
    ├── [忽略] 考试提示
    ├── ✅ 正文（统计）
    └── ✅ 选项（统计，去掉题号及 A-G 等标记）
    Part C（翻译）
    ├── [忽略] 考试提示
    └── ✅ 翻译正文（统计，去掉数字标记）

Section III 写作
    Part A（应用文）
    ├── [忽略] 考试提示
    └── ✅ 写作题目描述正文（统计，忽略数字）
    Part B（大作文）
    ├── [忽略] 考试提示
    └── ✅ 写作题目描述正文（统计，忽略数字）
```

### 3.3 内容分区标签

每段被统计的文本打上两个标签：

| 维度 | 可选值 |
|------|--------|
| **题型（exam_type）** | `完形填空` / `阅读` / `翻译` / `7选5` / `排序` / `选标题` / `写作` |
| **区域（section）** | `正文` / `选项` / `题目` |

不同题型的 section 取值：

| 题型 | 可能的 section |
|------|---------------|
| 完形填空 | `正文`、`选项` |
| 阅读 | `正文`、`选项`（题目选项） |
| 翻译 | `正文`、`题目` |
| 7选5 | `正文`、`选项` |
| 排序 | `正文`、`选项` |
| 选标题 | `正文`、`选项` |
| 写作 | `正文`（题目描述） |

## 四、单词统计标准（核心）

### 规则 1：常规变化 → 同一单词；不规则变化 → 不同单词

| 类型 | 例子 | 统计结果 |
|------|------|---------|
| 常规复数 | book / books | 统一为 `book` |
| 常规第三人称单数 | walk / walks | 统一为 `walk` |
| 常规过去式 | walk / walked | 统一为 `walk` |
| 常规现在分词 | walk / walking | 统一为 `walk` |
| 常规比较级/最高级 | tall / taller / tallest | 统一为 `tall` |
| 辅音双写 + 后缀 | big / bigger / biggest | 统一为 `big`（仍属常规） |
| **不规则**过去式 | run / ran | `run` 和 `ran` **分开统计** |
| **不规则**过去分词 | go / went / gone | `go`、`went`、`gone` **分开统计** |
| **不规则**比较级 | good / better / best | `good`、`better`、`best` **分开统计** |
| **不规则**复数 | child / children | `child` 和 `children` **分开统计** |
| **不规则**复数 | mouse / mice | `mouse` 和 `mice` **分开统计** |

**实现方案：**
- 建立**不规则形式词典**（几百个词条），涵盖常见的不规则动词、不规则名词复数、不规则形容词比较级
- 对于不在不规则词典中的单词，使用 NLTK WordNet Lemmatizer 进行词形还原，结果作为规范形式
- 对于不规则词典中的形式，保留其原始拼写作为独立条目

### 规则 2：不同词性 — 拼写相同→同一单词；拼写不同→不同单词

| 例子 | 统计结果 |
|------|---------|
| `run`（动词）+ `run`（名词，跑步） | 同一单词 `run` |
| `beauty` + `beautiful` | 不同单词（拼写不同） |

### 规则 3：忽略长度小于 3 的单词

如 `a`、`an`、`at`、`to`、`in`、`on`、`be`、`is`、`am`、`he`、`it`、`we`、`us`、`my`、`me`、`no`、`so`、`or`、`if`、`do`、`go`、`of` 等均不统计。

### 规则 4：缩写不计入统计

包含撇号（`'`）的缩写形式全部排除：`don't`、`can't`、`won't`、`it's`、`he's`、`they're`、`I'm`、`you'll` 等。

### 规则 5：专有名词不计入统计

人名、姓氏、机构名、地名均不统计。如 `Smith`、`John`、`Microsoft`、`NASA`、`America`、`London` 等。

**实现方案（多层过滤）：**
1. 维护一个专有名词黑名单（常见人名、地名、机构名）
2. 利用 NLTK NER（命名实体识别）在句子上下文中识别
3. 首字母大写启发式：如单词首字母大写且不在句首且不在常见普通名词词典中 → 大概率是专有名词

## 五、数据库设计

```sql
-- 单词主表
CREATE TABLE words (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word TEXT NOT NULL UNIQUE,        -- 规范形式（小写）
    total_freq INTEGER DEFAULT 0,     -- 总词频（所有年份、题型合计）
    year_count INTEGER DEFAULT 0,     -- 出现年份数
    first_year INTEGER,               -- 首次出现年份
    last_year INTEGER,                -- 最后出现年份
    freq_by_year TEXT,                -- JSON: {"2000": 3, "2001": 5, ...}
    freq_by_type TEXT                 -- JSON: {"完形填空": 10, "阅读": 20, ...}
);

-- 单词出现位置明细表
CREATE TABLE occurrences (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    surface_form TEXT,                -- 原文中的实际拼写形式
    year INTEGER NOT NULL,            -- 年份
    exam_type TEXT NOT NULL,          -- 题型：完形填空/阅读/翻译/7选5/排序/选标题/写作
    section TEXT NOT NULL,            -- 区域：正文/选项/题目
    sentence TEXT NOT NULL,           -- 所在完整句子（已清理数字/标记）
    word_offset INTEGER,              -- 单词在句子中的字符起始位置（用于前端高亮定位）
    word_length INTEGER,              -- 单词长度
    FOREIGN KEY (word_id) REFERENCES words(id)
);

CREATE INDEX idx_words_word ON words(word);
CREATE INDEX idx_words_freq ON words(total_freq DESC);
CREATE INDEX idx_words_yearcount ON words(year_count DESC);
CREATE INDEX idx_occ_word ON occurrences(word_id);
CREATE INDEX idx_occ_year ON occurrences(year);
CREATE INDEX idx_occ_type ON occurrences(exam_type);
CREATE INDEX idx_occ_word_year ON occurrences(word_id, year);
```

## 六、后端 API 设计

### 6.1 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/words` | 获取词频列表（支持全部筛选参数组合） |
| GET | `/api/words/<word>` | 获取单个词的统计详情 |
| GET | `/api/words/<word>/occurrences` | 获取单词的所有出现位置（分页，每页10条） |
| GET | `/api/meta/filters` | 获取筛选器可选值范围 |
| GET | `/api/export` | 导出当前筛选结果为 Excel |

### 6.2 筛选参数（全部可组合）

```
GET /api/words?
  freq_min=5                  # 最小总词频
  &freq_max=100               # 最大总词频
  &years=2020,2021,2022       # 出现在这些年份中的单词（逗号分隔，或的关系）
  &year_count_min=2           # 最少出现年份数
  &year_count_max=5           # 最多出现年份数
  &first_year=2020            # 首次出现年份（等于该年或之后）
  &exam_types=阅读,完形填空    # 出现在这些题型中的单词（逗号分隔，或的关系）
  &search=china               # 单词搜索（模糊匹配）
  &page=1                     # 页码
  &page_size=50               # 每页数量
  &sort_by=freq               # 排序字段：freq / year_count / first_year / word
  &sort_order=desc            # desc / asc
```

### 6.3 单词出现位置分页

```
GET /api/words/<word>/occurrences?page=1&page_size=10
```

返回以卡片为单位的出现位置列表，每页 10 个卡片。每个卡片包含：
- 年份、题型、区域标签
- 完整句子文本
- 单词在句子中的偏移量（用于前端高亮）
- 如果是完形填空选项，额外附带本题的题目句和四个选项

### 7.4 卡片展示规则（按题型 + 区域区分）

| 题型 | 区域 | 卡片显示内容 |
|------|------|-------------|
| 完形填空 | 正文 | 包含该单词的**完整句子**，单词高亮（如有多处则全部高亮） |
| 完形填空 | 选项 | **题目挖空句** + **四个选项**，单词在出现处高亮 |
| 阅读 | 正文 | 包含该单词的**完整句子**，单词高亮 |
| 阅读 | 选项 | 包含该单词的**完整句子**（题目或选项句），单词高亮 |
| 7选5 | 正文 | 包含该单词的**完整句子**，单词高亮 |
| 7选5 | 选项 | **仅显示选项这一句话**，单词高亮 |
| 排序 | 正文 | 包含该单词的**完整句子**，单词高亮 |
| 排序 | 选项 | **仅显示选项这一句话**，单词高亮 |
| 选标题 | 正文 | 包含该单词的**完整句子**，单词高亮 |
| 选标题 | 选项 | **仅显示选项这一句话**，单词高亮 |
| 翻译 | 正文 | 包含该单词的**完整句子**，单词高亮 |
| 翻译 | 题目 | **题目原句**，单词高亮 |
| 写作 | 正文 | 包含该单词的**完整句子**，单词高亮 |

### 7.5 高亮规则

- 卡片展示的句子中，目标单词**每次出现**都需要高亮
- 如果同一个单词在一句话中出现 3 次，则 3 处都要高亮
- 高亮样式：黄色背景 + 加粗

- 导出当前筛选条件下的词频数据
- Excel 文件包含：
  - **Sheet 1「词频统计」**：单词、总词频、出现年份数、首次年份、最后年份、分年词频明细（每年一列）、分题型词频明细（每个题型一列）

## 十一、关键风险与应对

| 风险 | 应对策略 |
|------|---------|
| PDF 是扫描版（图片）而非文字版 | 先尝试 pdfplumber 文本提取；如为扫描件需引入 OCR（tesseract + 中英文语言包） |
| 题型边界识别不准确 | 先抽样 2000、2010、2020 三年手动标注边界，作为测试用例验证 |
| 不规则形式词典覆盖不全 | 先覆盖最常见的 200-300 个不规则形式，后续根据统计结果补充 |
| 专有名词误杀/漏杀 | 多层过滤 + 保留人工检查入口；将疑似专有名词的单词标记后人工确认 |
| 句子切分不准确 | 以 `. ? !` 为主边界，结合空格+大写字母的英文句首模式辅助判断 |
| 完形填空选项聚类错误 | 以连续 ABCD 标记行作为选项组的识别特征 |
| 大词表前端性能 | 后端分页 + 数据库索引优化 |
